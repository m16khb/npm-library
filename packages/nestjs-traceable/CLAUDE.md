# nestjs-traceable

## WHAT: 기술 스택

- **TypeScript**: 5.7+ (ES2022)
- **의존성**:
  - @nestjs/common ^10.0.0 || ^11.0.0 (peer)
  - @nestjs/core ^10.0.0 || ^11.0.0 (peer)
  - rxjs ^7.0.0 (peer)

## WHY: 존재 이유

NestJS 애플리케이션에서 메서드 호출 추적과 로깅을 손쉽게 구현하기 위함. AOP(관점 지향 프로그래밍) 패턴을 활용하여 비즈니스 로직과 추적 로직을 분리하여 코드의 가독성과 유지보수성을 향상시킴.

## HOW: 주요 기능

### 1. Traceable 데코레이터
- 메서드 실행 시작/종료/예외 자동 로깅
- 실행 시간 측정
- 선택적 인자/결과 로깅
- 커스텀 작업 이름 지정

### 2. TraceableModule
- 전역 설정 관리
- 커스텀 로거 주입
- 모듈 수준의 추적 설정

### 3. 인터셉터 기반 구현
- NestJS 인터셉터를 활용한 실행 흐름 제어
- 메서드 메타데이터 활용
- 예외 처리 및 재전파

### 4. 유연한 로깅 옵션
- 다양한 로그 레벨 지원 (debug, info, warn, error, verbose)
- 사용자 정의 로거 구현 가능
- 예외 로깅 선택적 활성화
